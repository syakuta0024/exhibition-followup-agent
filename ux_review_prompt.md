# UXレビュー依頼：ユーザー目線での使用感チェック

**重要：このタスクはコードの変更を一切行わないでください。**
**分析・指摘・改善提案のみをリストアップしてください。**

## 依頼内容

このアプリ（展示会フォローアップAIエージェント）を
「展示会翌日に使う営業担当者」の目線で使用するシナリオを想定し、
UX上の問題点・改善提案をレビューしてください。

## 想定ユーザー像

- 職種：製造業SIerの営業担当（30〜45歳）
- AIツールへの習熟度：中程度（ChatGPTは使ったことある程度）
- 状況：展示会翌日の朝、2000件のリードCSVを手元に持っている
- 目標：午前中にフォローメールを送り終えたい
- 心理：早く終わらせたい、失敗したくない、難しいことはしたくない

## レビューしてほしいファイル

- `app.py`（UIの全体フロー・サイドバー・各タブ）
- `src/config.py`（フィールド定義・ラベル名）
- `src/agent.py`（処理フロー・エラーメッセージ）

## チェックしてほしい観点（それぞれ具体的に指摘してください）

### 1. 初回起動時の迷子リスク
- 何をすればいいか最初に分かるか
- ボタンの押す順序が分かりにくい箇所はないか
- 「ナレッジベース構築」と「PDFをベクトルDBに追加」の違いが
  一見して分かるか

### 2. エラー・失敗時の対応
- エラーメッセージは日本語でわかりやすいか
- APIキー未設定・ファイル形式間違い等のよくあるミスに
  適切なガイダンスが出るか
- 途中で失敗した場合に、最初からやり直しになるか・
  続きから再開できるか

### 3. カラムマッピング画面
- 自動推定が外れたときの修正が直感的か
- 必須・任意の区別が視覚的に明確か
- 「確定」ボタンを押した後に戻れるか

### 4. メール生成の前後
- 生成中の待ち時間に何が起きているか分かるか
- 生成されたメールの品質が良いか悪いか、
  ユーザーが判断できる情報が提示されているか
- 編集して再生成したいときの操作が分かるか

### 5. 一括生成時のリスク
- 2000件一括生成したときに問題が起きた場合の対処が分かるか
- 進捗が途中で止まったとき、ユーザーはどう対応するか
- コスト（APIコスト）への言及・警告はあるか

### 6. データプライバシーへの懸念
- 顧客情報・CRM情報をアップロードすることへの
  説明や注意書きはあるか
- ダミーデータと実データの混在リスクについての案内があるか

### 7. 用語・ラベルの分かりやすさ
- 「ナレッジベース」「チャンク」「ベクトルDB」等の
  技術用語が非エンジニアに伝わるか
- ランクA〜Eの意味がUIで説明されているか
- 「CTA」という略語は分かるか

### 8. モバイル・画面サイズへの対応
- サイドバーが多すぎてメインコンテンツが狭くなっていないか
- 情報の優先度と画面上の配置が対応しているか

## 出力形式

以下の形式でまとめてください：

```
## 🔴 重大な問題（すぐ直すべき）
- [場所] 問題の説明
  → 改善案

## 🟡 中程度の問題（できれば直したい）
- [場所] 問題の説明
  → 改善案

## 🟢 軽微な改善提案（余裕があれば）
- [場所] 問題の説明
  → 改善案

## 💡 全体的な所感
（3〜5行で総評）
```

重要：実装は行わないでください。
問題点のリストアップと改善案の提示のみです。
