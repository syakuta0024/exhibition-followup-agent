# 展示会フォローアップAIエージェント — システム概要資料

**バージョン**: 2026年4月時点  
**対象読者**: 導入担当者・営業マネージャー・IT担当者

---

## 1. 背景と課題

### 展示会フォローアップの現状

製造業の展示会では1回あたり数百〜2,000件のリードを獲得する。しかし翌営業日に全員へパーソナライズされたメールを送ることは現実的に困難で、以下の問題が生じている。

| 課題 | 内容 |
|---|---|
| **時間** | 1件あたり5〜10分 × 2,000件 = 最大333時間（1人では不可能） |
| **品質のばらつき** | 担当者ごとにメールの質・スピードが異なる |
| **情報の散逸** | 当日の会話メモが活かされないまま定型文を送信 |
| **機会損失** | 反応が早いほど成約率が高いが、フォローが翌週以降になりがち |

### 音声録音という未活用資産

展示会ブースでの会話には、名刺やアンケートに書かれない重要情報が含まれている。

- 「今期中に予算を確保したい」（予算感・タイムライン）
- 「現場の課長は乗り気だが役員承認が必要」（決裁プロセス）
- 「競合の○○も検討している」（競合状況）

これらは従来、担当者の記憶や断片的なメモにしか残らなかった。

---

## 2. 目的と解決アプローチ

### 目的

> 展示会翌日の午前中に、2,000件のリード全員へ個別最適化されたフォローアップメールを送れる状態にする。

### 解決アプローチ

```
【従来】
名刺 → 手入力 → 定型文コピペ → 修正 → 送信
（1件5〜10分 × 2,000件）

【このシステム】
CSVアップロード → AI自動処理 → 確認 → 送信
（1件8〜11秒 × 2,000件 ＋ 音声がある場合はニーズを優先反映）
```

作業削減率: 約 **60〜70%**

## 4. データの流れ

### 4-1. メール生成のコンテキスト優先順位

```
★ 最優先（音声あり時）
  音声録音 → 文字起こし → ニーズ抽出
  └ 課題・ニーズ・予算感・決裁者・温度感

① 展示会情報（全リード共通）
  └ 展示会名・開催日・会場

② リードデータ（来場者情報）
  └ 氏名・会社名・部署・役職・関心製品・商談メモ・独自アンケート

③ CRM商談履歴（過去接触がある場合）
  └ ライフサイクルステージ・最終接触日・担当者（HubSpot形式）

④ 製品技術資料（RAG検索）
  └ 関心製品に関連するMarkdown・PDF資料

⑤ 企業最新情報（Web検索）
  └ 事業内容・ニュース・プレスリリース（DuckDuckGo）
```

### 4-2. リードからメール生成までの流れ

```
リードCSV（Lead Manager / Q-PASS / Sansan等）
  ↓ アップロード + カラムマッピング
標準化DataFrame（visitor_name, company_name, email, lead_rank, memo ...）
  ↓
FollowUpAgent.process_lead()
  ├─ ランク正規化: "★5" → A, "3" → C 等
  ├─ RAG検索: 関心製品キーワードで製品資料を検索
  ├─ CRM照合: メール/会社名でHubSpotデータと紐づけ
  ├─ Web検索: 企業の事業内容・最新ニュースを取得
  ├─ 音声コンテキスト: 文字起こし + ニーズ抽出結果（あれば最優先）
  └─ EmailGenerator: 全コンテキストを統合してLLMでメール生成
  ↓
生成結果 {件名, 本文, 次のアクション, 参照資料, 品質スコア}
  ↓
CSVダウンロード
```

### 4-3. 音声紐づけの流れ

```
音声ファイル（20260424_営業A_001.mp3）
  ↓ ファイル名解析
担当者名「営業A」を抽出
  ↓ タイムスタンプ照合（メタデータ × QRスキャン時刻）
                ┌─ 時刻一致 → 🟢 自動確定
                ├─ 担当者名一致・時刻なし → 🟡 順番で仮紐づけ
                └─ 担当者名不明 → 🔴 手動選択
  ↓ 紐づけ確定
  ↓ Whisper API で文字起こし
  ↓ LLM でニーズ構造化抽出
  └─ メール生成時に最優先コンテキストとして注入
```

## 5. 必要なデータ・ファイル

### 5-1. 必須

#### OpenAI APIキー
- `.env` ファイルに `OPENAI_API_KEY=sk-...` を記載
- 使用モデル: `gpt-5.4-nano`（メール生成）、`text-embedding-3-small`（RAG）、`whisper-1`（音声文字起こし）

#### リードCSV
展示会管理システムからエクスポートしたCSVファイル。

| 対応システム | 備考 |
|---|---|
| Lead Manager（RX Japan） | 来場者スキャンデータ |
| Q-PASS | カスタム質問列も自動取込 |
| Sansan | 名刺データ |
| 自社タブレット | 任意のCSV形式に対応 |

必須カラム（カラム名はどんな表記でも自動推定）:

| フィールド | 説明 |
|---|---|
| 氏名 | 来場者の氏名 |
| 会社名 | 所属企業名 |
| メールアドレス | 送信先アドレス |

あると品質が上がるカラム:

| フィールド | 説明 |
|---|---|
| 評価・ランク | A〜E または 星1〜5（自動変換） |
| 商談メモ | 展示会での会話内容 |
| 関心製品 | 興味を示した製品名 |
| 担当者名 | ブース担当営業（音声紐づけに使用） |
| スキャン時刻 | QRスキャン時刻（音声紐づけ精度向上） |

---

### 5-2. 任意（品質向上に大きく貢献）

#### 製品技術資料（Markdown / PDF）
`data/tech_documents/` フォルダに配置、または画面からPDFをアップロード。

```
data/tech_documents/
  ├── 製品A_概要.md
  ├── 製品B_技術仕様.md
  └── サービスC_導入事例.md
```

| 形式 | 備考 |
|---|---|
| Markdown (.md) | テキストが直接読める形式が最適 |
| PDF | テキスト埋め込み型のみ対応（スキャン画像PDFは非対応） |

#### CRM CSV（HubSpot Contacts エクスポート）
過去の商談履歴があるとメールの文脈精度が大きく向上。

対応フィールド: `Email`, `Company name`, `Lifecycle stage`, `Lead status`, `Last activity date`, `Contact owner` 等

#### 音声録音ファイル
展示会当日の来場者との会話録音。

| 項目 | 推奨仕様 |
|---|---|
| 形式 | mp3 / wav / m4a |
| ファイルサイズ | 25MB以下（Whisper API制限） |
| 命名規則 | `YYYYMMDD_担当者名_連番.mp3` |
| マイク | 有線ラペルマイク（1,000〜3,000円）推奨 |

**ファイル命名例:**
```
20260424_営業A_001.mp3  ← 営業Aの1件目
20260424_営業A_002.mp3  ← 営業Aの2件目
20260424_営業B_001.mp3  ← 営業Bの1件目
```

## 6. 使い方（操作フロー）

### ステップ① データ読込

```
サイドバー:
  1. 送信元会社名を入力（例: 株式会社○○）
  2. CSVファイルをアップロード
  3. カラムマッピングを確認・調整
  4. 「✅ マッピング確定」をクリック
```

自動推定機能により、ほとんどのカラム名は自動でマッピングされます。  
必須フィールド（氏名・会社名・メールアドレス）が未設定の場合は確定できません。

---

### ステップ② ナレッジベース構築（任意・推奨）

```
サイドバー:
  1. 「🔨 ① ナレッジベース構築」をクリック
     → data/tech_documents/ のMarkdownを読み込み
  2.（任意）PDFをアップロードして「🔨 ② PDFを追加」
```

製品資料が登録されると、来場者の関心に合った製品情報が自動でメールに反映されます。  
CRM CSVがある場合もサイドバーからアップロードできます。

---

### ステップ③（任意）音声管理

```
「🎙️ 音声管理」タブ:

Step A: 音声ファイルをアップロード
  → アップロード直後に合計時間・文字起こし費用の概算を表示

Step B: 紐づけ確認
  → 🟢 自動確定 / 🟡 仮紐づけ（要確認） / 🔴 手動選択
  → 「✅ 紐づけを確定」をクリック

Step C: 文字起こし・ニーズ抽出
  → 「▶ X件を文字起こし・ニーズ抽出」をクリック
  → Whisper APIで文字起こし → LLMでニーズ構造化
  → 課題・ニーズ・予算感・決裁者・温度感を抽出
```

---

### ステップ④ メール生成

#### 個別生成（確認しながら）
```
「✉️ メール生成」タブ:
  1. 来場者をドロップダウンで選択
  2. 「✨ メール生成」をクリック
  3. 処理ステップ（ランク確認→資料検索→CRM照合→Web検索→生成）をリアルタイム確認
  4. 生成されたメールを確認・コピー
```

#### 一括生成（全件）
```
「✉️ メール生成」タブ → 「📧 全件一括生成」:
  1. コスト・時間の見積もりを確認
  2. 途中から再開したい場合は「▶️ 続きから実行」
  3. 最初からやり直す場合は「🔄 最初から実行」
  4. 処理中はブラウザを閉じないこと（閉じても生成済み分は保存済み）
```

---

### ステップ⑤ ダウンロード

```
「📊 生成履歴・ダウンロード」タブ:
  → 「📥 生成結果をCSVでダウンロード」
```

ダウンロードCSVの列:

| 列名 | 内容 |
|---|---|
| visitor_name | 氏名 |
| company_name | 会社名 |
| lead_rank | 商談確度（A〜E） |
| email_to | 送信先メールアドレス |
| subject | メール件名 |
| body | メール本文 |
| cta | 次のアクション |
| ref_tech_docs | 参照した技術資料 |
| ref_crm | 参照したCRM情報 |

## 8. 処理速度

| 条件 | 1件あたり | 100件 | 2,000件 |
|---|---|---|---|
| Web検索なし | 約8秒 | 約13分 | 約4.4時間 |
| Web検索あり | 約11秒 | 約18分 | 約6.1時間 |

**実用的な使い方**: 商談確度ABのみ先に処理（全体の20〜30%）→ 2〜3時間で完了。

## 9. 商談確度ランクの定義

| ランク | 意味 | メールのトーン |
|---|---|---|
| **A** | 最優良／確度高 | 具体的な次のアクションを提案（デモ・訪問日程） |
| **B** | 商談有望 | 課題解決への具体策を強調 |
| **C** | 要フォロー | 製品価値を丁寧に説明 |
| **D** | 興味あり | 情報提供メイン |
| **E** | 未評価 | 基本的なお礼メール |

## 10. プライバシーとデータ取り扱い

⚠️ このシステムは以下の情報を **OpenAI API に送信** します。

- 来場者の氏名・メールアドレス・会社名
- 商談メモ・アンケート回答
- 音声文字起こしテキスト（会話内容）
- CRM商談履歴

**利用前に以下を確認してください:**

1. 取り扱う顧客情報が貴社の情報セキュリティポリシーに照らして外部サービスへの送信が許可されているか
2. 展示会参加者への個人情報取り扱い同意の範囲内であるか
3. 機密・社外秘情報が含まれていないか

不明な場合はIT担当者または上長にご相談ください。

## 11. システム要件と起動方法

### 動作環境

- Python 3.10 以上
- `.venv` 仮想環境（依存パッケージ一式）
- OpenAI APIキー

### 起動手順

```bash
# 1. APIキーを設定（初回のみ）
# .env ファイルを作成して以下を記載:
OPENAI_API_KEY=sk-...

# 2. アプリを起動
.venv/Scripts/streamlit run app.py   # Windows
# または（.venv がアクティブな場合）
streamlit run app.py
```

ブラウザが自動で開きます（http://localhost:8501）。

### ディレクトリ構成（自社データ配置場所）

```
exhibition-followup-agent/
├── .env                        ← APIキーをここに記載
├── data/
│   ├── tech_documents/         ← 自社の製品資料MDを配置
│   └── crm_records/            ← （使用しない場合は空のまま）
└── chroma_db/                  ← ナレッジベースの保存先（自動生成）
```
