# 音声ファイル テストケース一覧

## テスト設計の前提

CSVの担当者パターン：
- test_leads_rx_standard / with_custom: 「営業A」「営業B」「営業C」
- test_leads_messy: 「Eigyo-A」「Eigyo-B」「Eigyo-C」

---

## ケース① 🟢 正常（自動確定されるべき）

ファイル名にYYYYMMDD + 担当者名 + 連番が全部ある。
CSVのscan_timeと±10分以内で一致する想定。

```
20260424_営業A_001.mp3   → 田中太郎（10:05）に紐づくはず
20260424_営業A_002.mp3   → 佐藤次郎（10:48）に紐づくはず
20260424_営業A_003.mp3   → 高橋健  （11:33）に紐づくはず
20260424_営業B_001.mp3   → 鈴木花子（10:32）に紐づくはず
20260424_営業B_002.mp3   → 山田五郎（11:15）に紐づくはず
20260424_営業C_001.mp3   → 伊藤由美（11:50）に紐づくはず
```

---

## ケース② 🟡 仮紐づけ（要確認になるべき）

担当者名はあるが日付がない → 順番で仮紐づけ。

```
営業A_001.mp3            → 営業Aの1件目（田中太郎？）
営業B_001.mp3            → 営業Bの1件目（鈴木花子？）
```

---

## ケース③ 🔴 未紐づけ（手動選択になるべき）

担当者名が不明なファイル名。

```
rec001.mp3               → 担当者不明
recording_001.mp3        → 担当者不明
audio001.mp3             → 担当者不明
```

---

## ケース④ ⚠️ 表記ゆれ（要注意）

CSVの「営業A」に対してファイル名が微妙に違う。
AudioMatcherが吸収できるか確認したいポイント。

```
20260424_営業Aさん_001.mp3   → 「営業A」と一致するか？
20260424_EigyoA_001.mp3     → messyCSV用（英語表記）
20260424_A担当_001.mp3       → 別表記でも拾えるか？
```

---

## ケース⑤ タイムスタンプ境界値（AudioMatcherの許容誤差±10分）

```
20260424_営業A_001.mp3
  ファイルメタ時刻: 10:05 → CSVスキャン時刻10:05 → ✅ 一致（誤差0分）

20260424_営業A_002.mp3
  ファイルメタ時刻: 10:10 → CSVスキャン時刻10:05 → ✅ 一致（誤差5分、許容内）
  ファイルメタ時刻: 10:20 → CSVスキャン時刻10:05 → ❌ 不一致（誤差15分、許容外）
  ※ 実際の録音ファイルはメタデータが録音開始時刻になるため、
     スキャン時刻との差が大きくなりやすい点に注意
```
