# UXレビュー結果：展示会フォローアップAIエージェント

**対象ユーザー**：製造業SIerの営業担当（展示会翌日・2000件リード・午前中に完了したい）  
**調査ファイル**：`app.py` / `src/config.py` / `src/agent.py`

---

| # | 場所 | 対応状況 |
|---|---|---|
| 1 | 一括生成 | ⬜ 未対応 |
| 2 | 一括生成 | ⬜ 未対応 |
| 3 | 一括生成 | ⬜ 未対応 |
| 4 | カラムマッピング | ✅ 対応済み |
| 5 | 初回起動 | ⬜ 未対応 |

### #1 2000件の処理時間が非現実的（約6時間）

8秒/件 + Web検索3秒 = **11秒 × 2000件 = 約366分**。コスト確認ダイアログに表示されるが、「午前中に終わらせたい」というユーザーゴールと根本的に矛盾する。

→ **改善案**：コスト確認ダイアログ内に商談確度フィルター（ABのみ等）を前面表示。「まず上位N件だけ生成しますか？」の提案を自動表示。

### #2 途中中断で全結果が消える・再開不可

「途中で止まった場合は再度実行してください（生成済み結果は消えます）」の1行のみ。ブラウザが落ちると全て消え、再実行でも同コストが再請求される。

→ **改善案**：生成結果を逐次 `session_state["results"]` に追記して途中再開を可能にする。最低限「中断時は即CSVダウンロードできます」という救済ガイドを表示。

### #3 エラー行の再生成手段がない

一括完了後「CSVの「件名」列が 'ERROR' の行を確認してください」だけ表示。UIからエラー行だけ再生成する方法がない。

→ **改善案**：完了後に「❌ エラーのXX件を再実行」ボタンを表示。例外メッセージを日本語化して表示。

### #4 必須フィールド未マッピングでも確定できる ✅

`st.warning` のみで `mapping_confirmed = True` が実行されていた。`st.error + return` で対応済み。

### #5 マッピング確定後に「次: ナレッジベース構築」の案内がない

ウェルカム画面の「使い方の流れ」に手順は書いてあるが、確定後の画面遷移では案内がなく、メール生成タブを開いて初めて気づく。

→ **改善案**：マッピング確定直後の画面上部に `st.info("✅ データ読み込み完了！次はサイドバーの「🔨 ナレッジベース構築」を押してください")` を表示。

### #6 「ナレッジベース構築」と「PDFをベクトルDBに追加」の違いが不明

両方に「🔨」ボタンが並ぶ。順序関係（構築→PDF追加）や「構築してもPDFは消えない」ことがUIで説明されていない。

→ **改善案**：各ボタンのhelpテキストに順序と説明を追記。番号（①②）で手順を明示。

### #7 技術用語がそのまま露出している

「ベクトルDB検索」「チャンク数」「インデックス未構築」等が処理ステップ・ナレッジベース確認タブ・エラーメッセージに表示される。

→ **改善案**：
- `ベクトルDB検索` → `製品資料の検索`
- `チャンク数` → `登録件数`
- `インデックス未構築またはクエリなし` → `製品資料がまだ登録されていません`

### #8 「CTA」という略語が非エンジニアに伝わらない

生成結果CSVのカラム名・UI上のラベルに "CTA" が使われている。

→ **改善案**：`CTA` → `次のアクション`（または `CTA（行動喚起）` と併記）

### #9 ランクA〜Eの定義がUIに存在しない

商談確度フィルター・ランク推定結果で A/B/C/D/E が使われるが、各ランクの意味説明が一切ない。

→ **改善案**：フィルターの `help` に「A: 最優良/確度高, B: 商談有望, C: 要フォロー, D: 興味あり, E: 未評価」を追加。

### #10 一括生成の進捗に残り時間の表示がない

`[1/2000]` のカウントはあるが残り時間の推定がなく、不安になる。

→ **改善案**：処理開始時刻を記録し、1件あたりの実績から `残り約XX分` を逐次計算して表示。

### #11 任意フィールドが4列グリッドで圧倒される

任意フィールド9個が均等に並び、どれを設定すれば品質が上がるか分からない。

→ **改善案**：`memo`（商談メモ）と `interested_products`（関心製品）を「推奨フィールド」として上部にグループ化。残りはexpanderに格納。

### #12 APIキー未設定エラーが非エンジニア向けでない

`OPENAI_API_KEY が設定されていません。.env ファイルに～` → `.env` ファイルの操作方法が分からない。

→ **改善案**：「APIキーが設定されていません。IT担当者に設定を依頼してください。」という形式に変更。

### #13 スキャンPDF非対応の事前告知がない
アップロード後にエラーが出る。→ アップロード欄のhelpに「スキャン画像のみのPDFは非対応（テキスト埋め込みPDFのみ）」を追記。

### #14 生成メールの品質を判断する情報が少ない
参照チャンクは表示されるが、品質スコアの基準説明がない。→ 「80点以上: 十分な情報あり / 60点以下: 情報不足の可能性」等の基準を付記。

### #15 リード一覧からワンクリックでメール生成できない
リード一覧タブ → メール生成タブ → ドロップダウンで再選択という2〜3ステップが必要。→ 一覧の各行に「📧 このリードのメールを生成」ボタンを追加。

### #16 デモデータ使用中であることが分かりにくい
実データと混同するリスクがある。→ サイドバートップに「🗂️ デモデータを使用中」バナーを常時表示。

### #17 サイドバーが6セクションで情報過多
初期表示からすべて展開されている。→ PDF・CRM・機能設定セクションをexpander（デフォルト折りたたみ）に変更。

### #18 展示会情報をカラムマッピング確定後に変更できない
変更するには「↩️ マッピングをやり直す」で戻る必要がある。→ サイドバーに展示会情報入力欄を常時表示。

### #19 ランク推定の「GPT 1回追加」のコスト感が不明
→ `約$0.00006/件追加` のように金額を添える。

## 💡 全体所感

プライバシー警告・自動カラムマッピング・コスト確認ダイアログ・処理ステップ可視化など配慮が随所にある良い実装。最大の課題は**一括生成の実用性**（2000件×11秒=6時間）で、「午前中完了」というユーザーゴールと相容れない。次点は**技術用語の露出**で、非エンジニアの営業担当が戸惑う原因になる。カラムマッピング・プライバシー通知は概ね良好で、細部の改善で十分使いやすくなる。
